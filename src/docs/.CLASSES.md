# Player Classes & Repercussions Reference

## Overview

Player classes gate which **repercussion types** a player can attach to their text additions during
collaborative writing phases. A player's class is chosen during setup and stored as a
`PlayerClassOption` on their `Player` object.

---

## Player Classes

| Class | Description | Unlocked RepercussionType |
|---|---|---|
| **Bard** | Spreads story outcomes to all players, including yourself | `ALL_PLAYERS` |
| **Herald** | Gives new titles to other players | `TITLE` |
| **Scribe** | Records histories and adds traits to other players | `TRAIT` |
| **Fabulist** | Converts encounters players survive into companion traits | `COMPANION` |

Defined in `model/PlayerClass.java` (enum). Serialized to the client as `PlayerClassOption`
(a plain class with the same fields) via `GET /player/classes`.

---

## Repercussion Types

| Type | Color | Effect |
|---|---|---|
| `TRAIT` | `#0288d1` (blue) | Gives a named trait to players at the story's encounter |
| `TITLE` | `#7b1fa2` (purple) | Gives a title; appended to `Player.displayName` |
| `COMPANION` | `#E60000` (red) | Converts the encounter label into a companion trait on those players |
| `ALL_PLAYERS` | `#f57c00` (orange) | Spreads TRAIT/TITLE/COMPANION effects to every player, not just encounter players |
| `SEQUEL` | `#388e3c` (green) | Tags the outcome as a sequel setup (effect TBD) |

Defined in `model/RepercussionType.java` (enum). Each type carries: `name`, `label` (input prompt
text), `instruction` (toggle-off description), `description` (toggle-on description), `color`.

## Data Flow

### 1 — Class selection

`GET /player/classes` → streams all `PlayerClass` enum values → serialized as `PlayerClassOption[]`
→ player picks one → stored on `Player.playerClass`.

### 2 — What repercussions are available to this player

`GET /collaborativeText/repercussionTypes?gameCode=X&authorId=Y`
→ `CollaborativeTextHelper.getPlayerRepercussionTypes`
→ reads `player.getPlayerClass().getRepercussionTypes()` (list of type name strings)
→ returns matching `RepercussionTypeOption[]`
→ player-client `repercussion-toggle` renders only these options.

### 3 — Attaching a repercussion to an addition

When a player makes an addition (`submitTextAddition`), the `TextAddition` may carry a
`Repercussion` object with `repercussionType` (enum name) and `repercussionSubmission` (the
trait/title text the player typed).

### 4 — Repercussion resolution on winner calculation

When a submission wins (`calculateWinningSubmission`), its additions are scanned for repercussions.
Collected repercussions are attached to the winning `OutcomeFork.repercussions`, then
`handleRepercussions` fires:

```
repercussions present?
  │
  ├── TRAIT or TITLE → handleNewTraitRepercussions
  │     applies Trait objects to players in story.playerIds
  │
  ├── COMPANION → adds Trait(encounterLabel, COMPANION) to story.playerIds
  │
  └── ALL_PLAYERS → handleSpread
        re-applies all traits above to every player in the game
        (not just the encounter's assigned players)

no repercussions AND roundNumber < 2?
  → auto-generates a SEQUEL-style "encounter again" default repercussion
    so the encounter has narrative weight even without explicit class action
```

### 5 — Traits on Player

Both `TRAIT` and `TITLE` repercussions become `Trait` objects appended to `Player.traits`.
`TraitType.TITLE` entries are rendered in `Player.getDisplayName()` (appended after a comma).
`TraitType.COMPANION` entries appear as companion badges in the player-client UI.

---

## Where Each Class Shows Up in the UI

- **Scribe / Herald**: `repercussion-toggle` shows a text input for the trait/title value
  (`label` field = "Add player trait" / "Add title").
- **Bard**: toggle shows the spread description; no text input needed (`label` = `""`).
- **Fabulist**: toggle shows the companion description; the encounter label is used automatically
  as the companion name (`label` = `""`).

## Key Files

| File | Role |
|---|---|
| `model/PlayerClass.java` | Enum: class names, descriptions, and which RepercussionTypes they unlock |
| `model/RepercussionType.java` | Enum: all repercussion types with display metadata and colors |
| `model/Repercussion.java` | Runtime object: `repercussionType` + `repercussionSubmission` carried on a TextAddition |
| `model/RepercussionTypeOption.java` | DTO serialized to the player-client (mirrors RepercussionType fields) |
| `model/PlayerClassOption.java` | DTO serialized to the player-client (mirrors PlayerClass fields) |
| `helper/CollaborativeTextHelper.java` | `getPlayerRepercussionTypes`, `handleRepercussions`, `handleNewTraitRepercussions`, `handleSpread` |
| `controller/PlayerController.java` | `GET /player/classes` |
| `controller/CollaborativeTextController.java` | `GET /collaborativeText/repercussionTypes` |
